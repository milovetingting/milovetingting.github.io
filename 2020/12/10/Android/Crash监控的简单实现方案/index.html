Crash监控的简单实现方案

前言

本文从Java层及Native展开,简单记录Android中的Crash监控方案。

Java层Crash

Java层的crash监控，可以通过实现Thread.UncaughtExceptionHandler接口，重写uncaughtException方法来实现，简单示意代码如下:

java

新建一个CrashUtil类，封装CrashHandler

java

在Application的onCreate方法中，初始化CrashHandler

java

在AndroidManifest中配置Application的name属性

xml

Native层Crash

Native层crash监控，可以采用Google的BreakPad开源框架。开源地址:https://github.com/google/breakpad

BreakPad需要编译后才能使用。

  1. 新建一个module，专门编译BreakPad。

  2. 要新建的breakpadlib的src/main目录下，新建cpp目录，这个目录名称随意。

  3. 在cpp目录下新建breakpad目录，这个目录名称随意。

  4. 将下载回来的BreakPad代码解压后，将里面的src目录都复制到刚才新建的breakpad目录下。

  5. 在breakpad目录下新建CMakeLists.txt文件，在里面配置如下

    cmake

  6. 在breakpadlib模块的src/main/java/包名/目录下，创建NativeCrash.java文件，在这里声明Native方法，方便其它模块调用

    java

  1. 在cpp目录下，新建breakpad_lib.cpp文件，实现native方法，并配置Native层crash的监听

    cpp

  2. 在cpp目录下创建CMakeLists.txt文件

    cmake

  3. 在breakpadlib的build.gradle文件中配置ndk

    gradle

  4. build这个module，就会生成包含so的aar，可以给其它模块引用，这样就可以为Native层的Crash增加监控。

breakpadlib目录结构如图

注意事项

在编译时，一般会报错。这是由于缺少相应的文件导致的。将linux_syscall_support.h文件放到cpp/breakpad/src/third_part/lss目录下，再编译就可以了。

app模块引用breakpad编译出来的aar

将编译出来的aar放到app模块的libs目录下，并配置app的build.gradle文件

gradle

同步工程，然后在前面创建的CrashUtil中增加对Native的Crash的初始化。

java

模拟Native层Crash

  1. 新建一个module:crash,用于生成演示Native层Crash所用的so。

  2. 在crash模块下的src/main/java/目录下创建CrashTest.java，声明native方法

    java

  3. 在crash模块下的src/main/目录下，新建jni目录

  4. 在jni目录下新建cpp目录，在cpp目录下创建crash_test.cpp

    cpp

  5. 生成so

    可以通过ndk-build直接生成so文件。在jni目录下创建Android.mk

    mk

    然后在jni的目录下，执行cmd命令ndk-build。如果没有在系统环境变量中配置ndk，一般会报'ndk-build' 不是内部或外部命令，也不是可运行的程序。通过Android Studio下载的ndk，一般在Android Studio配置的SDK目录下。进入SDK目录下的ndk目录，选择一个版本的ndk，将这个路径配置到环境变量中。重新打开CMD窗口，并定位到jni目录下，重新执行ndk-build,就会在jni目录下生成libs和obj目录。

    也可以在crash模块的build.gradle中配置来生成aar

    gradle

    在jni目录下创建CMakeLists.txt

    cmake

    build后，就可以生成包含so的aar文件

    crash模块的目录结构

  6. 将aar放到app模块的libs目录，并在build.gradle中增加对新加入的aar的引用

    gradle

测试Crash

Java Crash

在Activity的onCreate中加入以下代码

java

打包APK，安装到手机，运行APK，程序Crash。查看/sdcard/Android/data/包名/，发现在java_crash目录，生成了记录crash信息的文件。

Native Crash

在Activity的onCreate中加入以下代码

java

打包APK，安装到手机，运行APK，程序Crash。查看/sdcard/Android/data/包名/，发现在native_crash目录，生成了记录crash信息的文件xxx.dmp。

将dmp文件导出到电脑。然后在Android Studio的安装目录/bin/lldb/bin中，执行CMD命令

cmd

在当前目录下会生成crash.txt文件

Code

这个文件列出了操作系统、CPU等信息。

在ndk目录下，找到ndk\21.0.6113669\toolchains\arm-linux-androideabi-4.9\prebuilt\windows-x86_64\bin\目录，这里的arm-linux-androideabi-4.9具体以手机的CPU架构为准。

执行CMD

cmd

这里的libcrash_text.so就是发生crash的so文件，0x75c为上面转换出来的crash.txt中发生crash的地址。通过addr2line工具，可以看到发生crash的具体行数。
